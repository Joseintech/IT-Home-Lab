# Group Policy Management Lab

## Objective
Learn to configure and manage Group Policy Objects (GPOs) in a Windows Active Directory environment to enforce security settings, organizational rules, and administrative control across users and computers.

## Tools & Equipment
	•	AWS Windows Server instance with Active Directory installed
	•	Windows 11 client VM (or RDP access to AWS server)
	•	Internet connection

## Steps Performed
	1	Accessed the Domain Controller and opened Group Policy Management Console (GPMC).
	2	Created Organizational Units (OUs) for IT, HR, Finance, and other departments.
	3	Configured Group Policy Objects for each OU to enforce specific policies (e.g., password restrictions, desktop settings).
	4	Linked GPOs to the respective OUs.
	5	Tested policy application by creating sample users and observing applied settings.
	6	Verified successful Group Policy propagation and enforcement across OUs.

## Skills Practiced
	•	Group Policy creation, linking, and management
	•	OU organization and management
	•	Security policy enforcement across multiple users
	•	Technical documentation and lab organization

## Outcome
Successfully created and applied GPOs across multiple OUs in an Active Directory environment, demonstrating the ability to enforce security and organizational policies at scale. This lab provides a foundation for real world IT administration, policy management, and help desk scenarios.
