# Active Directory (CloudBased) Lab – AWS

## Objective
Deploy and configure a cloud hosted Active Directory environment to gain hands on experience with domain services, user management, and enterprise identity infrastructure commonly used in real world IT and help desk roles.

## Environment
	•	Cloud Provider: Amazon Web Services (AWS)
	•	Compute: EC2 Windows Server
	•	Operating System: Windows Server (Domain Controller)
	•	Directory Service: Active Directory Domain Services (AD DS)
	•	Access Method: Remote Desktop Protocol (RDP)

## Tools & Technologies
	•	AWS EC2
	•	Windows Server
	•	Active Directory Domain Services
	•	DNS
	•	Remote Desktop (RDP)

## Steps Performed
	1	Provisioned a Windows Server EC2 instance in AWS.
	2	Connected to the server securely using RDP.
	3	Installed the Active Directory Domain Services (AD DS) role.
	4	Promoted the server to a Domain Controller.
	5	Configured a new Active Directory forest and domain.
	6	Installed and configured DNS as part of the domain setup.
	7	Verified successful domain controller promotion and system reboot.
	8	Confirmed Active Directory services were running and accessible.

## Skills Demonstrated
	•	Active Directory fundamentals
	•	Domain controller deployment
	•	DNS integration with Active Directory
	•	Cloud-based Windows Server administration
	•	Remote system management (RDP)
	•	Troubleshooting installation and configuration issues
	•	Professional IT documentation

## Outcome
Successfully deployed a fully functional cloud-hosted Active Directory domain controller in AWS. This environment mirrors enterprise identity infrastructure and provides a foundation for advanced labs, including:
	•	User and group management
	•	Organizational Units (OUs)
	•	Group Policy Objects (GPOs)
	•	Help desk account administration scenarios
	•	Security and access control testing

This project demonstrates hands on experience with enterprise identity systems in a cloud environment, a critical skill for help desk, system administration, and entry level IT roles.
